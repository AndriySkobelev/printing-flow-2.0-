Проаналізуй поточні git-зміни і згенеруй опис для коміту.

Кроки:
1. Запусти `git diff --staged` — якщо нічого немає, запусти `git diff HEAD` щоб побачити всі незакомічені зміни.
2. Запусти `git status` щоб побачити нові файли.
3. Проаналізуй зміни і склади опис коміту:
   - Перший рядок: короткий заголовок у стилі conventional commits (feat/fix/refactor/chore тощо), до 72 символів, англійською
   - Якщо потрібно — короткий bullet-list того що змінилось (2-5 пунктів), теж англійською
   - Зроби окремі блоки для короткого опису та для bullet-list щоб міг окремо копіювати
4. Виведи готовий опис у code block щоб можна було скопіювати.

Не роби коміт сам — тільки сформуй текст опису.
